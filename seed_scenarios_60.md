# Alluvi — Seed the `scenarios` Catalog (60 curated)

Fills every column of `public.scenarios` for all 60 curated scenarios (the original 30 + 30 new all-held ones), including `composed_attributes`.

## Before you run

1. Run the **Learning Layer migration v2** first — the `scenarios` table must already exist.
2. Paste the SQL below into the **Supabase SQL editor** and run it once.
3. It is **idempotent**: `ON CONFLICT (scenario_id) DO UPDATE` means re-running it updates existing rows instead of erroring, so it is safe to re-run after edits.

## What each column gets

- `index_no` — 1..60 in catalog order (drives processing order; not the engine trigger).
- `scenario_id` — the stable id (e.g. `beach_sunrise_hold_31`).
- `tenant_id` — `NULL` (curated scenarios are shared by every tenant).
- `source` — `'curated'`.
- `category`, `difficulty`, `scenario_title` — denormalised for filtering/readability.
- `content` — the full scenario JSON (`scene`, `outfit`, `pose`, `grip_or_placement`, ...).
- `composed_attributes` — the canonical learning tags (`archetype, scene_category, environment, difficulty, has_phone, product_hand, hold_type, grip_level, box_orientation, framing, camera_height, lighting_type, time_of_day, mirror, fabric_family`).
- `version` — `'v1'`; `content_hash` — md5 of `content` for change-detection; `active` — `true`.
